# Repository Guide

The **FAEP Academy** is designed as a **GitHub-native engineering learning platform**.

Rather than storing every learning resource in a single repository, the Academy organizes content across multiple specialized repositories that work together as a unified engineering ecosystem.

As the FAEP ecosystem evolves, additional repositories can be integrated without changing the learner experience.

# Current Repositories

## FAEP Education

Primary engineering learning repository.

Repository contents include:

- Tutorials
- Hands-on Labs
- Jupyter Notebooks
- Engineering Projects
- Case Studies
- References
- Templates
- Samples
- Professional Development
- Student Learning Tracks
- Research Intelligence

Purpose

Provide structured engineering education and self-paced learning.

---

## FAEP Client Lab

Applied engineering repository.

Repository

https://github.com/vijaymohire/faep-client-lab

Purpose

Provide practical engineering experiences through:

- Engineering SDKs
- Engineering Providers
- Datasets
- Dashboards
- Templates
- Execution Environments
- Client Engineering Workflows
- End-to-End Engineering Examples

The Client Lab complements Academy learning by enabling learners to apply engineering concepts in realistic scenarios.

# Repository Organization Principles

Every repository should have:

- Clear purpose
- Consistent navigation
- Standard README
- Learning objectives
- Engineering resources
- Cross-repository links
- Contribution guidelines

This provides learners with a consistent experience regardless of which repository they are exploring.
