# 1. Purpose

This document defines the navigation standards for all educational assets within the FAEP Academy.

The navigation framework enables learners, educators, contributors, AI systems, and automation tools to efficiently discover, traverse, and manage learning resources.

Navigation should be:

- Consistent
- Predictable
- Reusable
- Metadata-driven
- Knowledge Graph enabled
- GitHub friendly

---

# 2. Scope

This standard applies to:

- Repository README files
- Courses
- Tutorials
- Notebooks
- Labs
- Projects
- References
- Learning Paths
- Career Paths
- Architecture Documents
- Standards
- Templates

---

# 3. Navigation Principles

Navigation shall:

- Be consistent throughout the repository.
- Minimize navigation depth.
- Support learner progression.
- Support Knowledge Graph traversal.
- Support AI-assisted recommendations.
- Support GitHub rendering.
- Avoid broken links.
- Encourage content discovery.

---

# 14. Quality Requirements

Navigation should

- Be consistent.
- Be easy to understand.
- Avoid circular references.
- Avoid dead links.
- Support GitHub rendering.
- Scale to large repositories.
- Support future automation.

---

# 15. Naming Conventions

Navigation labels should use consistent terminology.

Examples

| Preferred | Avoid |
|-----------|-------|
| Course | Class |
| Tutorial | Lesson |
| Notebook | Script |
| Lab | Exercise |
| Project | Assignment |
| Portfolio | Collection |

# 16. Repository Scalability

Navigation should remain effective as the Academy grows.

Future growth may include

- Hundreds of courses
- Thousands of tutorials

- Thousands of projects

Navigation should remain discoverable and maintainable at scale.
